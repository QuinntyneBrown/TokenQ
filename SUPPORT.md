# Support

Use the right channel so issues stay actionable.

## Questions

Open a GitHub issue with the `question` label. Include the command you ran,
your OS, and the .NET SDK version from:

```powershell
dotnet --info
```

## Bugs

Open a bug report and include:

- TokenQ version or commit.
- Operating system.
- .NET SDK/runtime version.
- Full command line.
- Expected behavior.
- Actual behavior.
- Minimal reproduction directory or command sequence.

## Feature Requests

Open a feature request with the problem first, then the proposed behavior.
Changes that affect generated output should also update `docs/specs/L2.md`.

## Security

Do not report vulnerabilities in public issues. Follow
[SECURITY.md](SECURITY.md).

## Response Expectations

TokenQ is maintained as an open source project. Response times may vary.
Well-scoped reports with reproduction steps are the easiest to review.
